\section{Architectural overview}
The system can be divided into four layers. 
%TODO: Improve wording why layers are needed.
\begin{enumerate}
    \item The presentation layer
    \item The visualization layer
    \item The parsing layer
    \item The data layer
\end{enumerate}
% Presentation Layer
The \textbf{presentation} Layer is responsible for interactions and rendering. It implements a split-view displaying the datatable and flow-graph simultaneously. 
%TODO: Explain why Split-View ==> what is called the locality i.e distance of context ??
This layer only presents the data from the state layer and dispatch ssers interactions. It does not directly manipulate the data.
% State Layer
The second layer, the \textbf{visualization layer}, is responsible for bridging the gap between the underlying formula structure and the flow graph visualization. The \textbf{parsing} layers only responsibility is to provide two conversions one from formula strings into an AST and the reversed direction. The foundation of the system is the \textbf{data layer} where the calculation engine of the spreadsheet resides and manages all the spreadsheet data.

For the Spreadsheet to flow graph direction, first the parser translates an unstructured formula String into an AST\footnote{Abstract Syntax Tree}. To bridge the gap between AST and flow graph we need another transformer which maps elements of the AST to elements of the flow graph. The use of an intermediate AST was deliberately chosen over raw string manipulation to ensure correctness when handling complex and deeply nested formulas for a very small performance cost.
